## 2024-05-18 - Sandbox iframe Code Injection XSS Vulnerability
 **Vulnerability:** Unsanitized user code was directly interpolated into the `srcdoc` HTML attribute of a sandboxed iframe using `${code}`. If a user provided `</script>`, it prematurely closed the script tag, allowing arbitrary HTML/JS execution. Furthermore, valid syntax errors in user code silently crashed the iframe setup instead of properly getting handled and passed back to the frontend UI via postMessage.
 **Learning:** This vulnerability existed because the developer did not escape the user input when injecting into a string intended to be evaluated inside an HTML `<script>` block. Using `encodeURIComponent(code)` is insufficient because it does not encode single quotes (`'`), causing valid JS code (e.g. `console.log('hi')`) to fail compilation.
 **Prevention:** When injecting dynamically generated user code as a string literal into a script, safely serialize the payload using `JSON.stringify(code)` and encode `<` to `\u003c` via `.replace(/</g, '\\u003c')`. This ensures accurate and secure escaping without breaking syntax, allowing proper runtime evaluation via `eval()`.
