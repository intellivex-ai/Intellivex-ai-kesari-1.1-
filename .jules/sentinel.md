## 2026-04-15 - [Fix IDOR in api/generate-image.ts]
**Vulnerability:** Insecure Direct Object Reference (IDOR) where any authenticated user could insert image messages into any other user's chat.
**Learning:** The application was missing explicit authorization checks (resource ownership verification) on the server side when performing operations via API endpoints that accept identifiers (like `chatId`) from the client.
**Prevention:** Always verify resource ownership (e.g., query the `chats` table to match the `user_id` with the authenticated `userId`) before allowing modifications or creations on that resource.
## 2024-05-28 - Missing Authorization/IDOR on Dev Server API
**Vulnerability:** The local development API endpoints (`/api/chat`, `/api/generate-image`, `/api/messages`) handled via `vite-api-plugin.ts` used `SUPABASE_SERVICE_ROLE_KEY` via `sbFetch`, effectively bypassing RLS. However, they lacked manual authorization checks to ensure the requested `chatId` or `chat_id` belonged to the authenticated `userId`. An attacker could perform actions on behalf of another user by providing their chat ID.
**Learning:** The dev server uses a proxy with service role privileges, making explicit programmatic resource ownership verification mandatory, mimicking the RLS protection that standard endpoints have.
**Prevention:** Always verify resource ownership (e.g., verifying `user_id` matches `userId`) on all CRUD API endpoints handling sensitive user resources when using an administrative database client. Reject unauthenticated or unauthorized requests with 401 or 403 explicitly rather than silently falling through.
