## 2026-04-15 - [Fix IDOR in api/generate-image.ts]
**Vulnerability:** Insecure Direct Object Reference (IDOR) where any authenticated user could insert image messages into any other user's chat.
**Learning:** The application was missing explicit authorization checks (resource ownership verification) on the server side when performing operations via API endpoints that accept identifiers (like `chatId`) from the client.
**Prevention:** Always verify resource ownership (e.g., query the `chats` table to match the `user_id` with the authenticated `userId`) before allowing modifications or creations on that resource.
## 2026-05-11 - Fix IDOR in vite-api-plugin.ts\n**Vulnerability:** Insecure Direct Object Reference (IDOR) where any authenticated user could modify or append data to any other user's chat or messages by manipulating the `chatId`.\n**Learning:** The dev server plugin uses `SUPABASE_SERVICE_ROLE_KEY` which bypasses Row Level Security (RLS). Consequently, explicit resource ownership verification must be done on the server before acting on the request.\n**Prevention:** Always explicitly fetch the resource's `user_id` from the database and verify it matches the authenticated `userId` before processing modifications on endpoints handling identifiers from the client.
