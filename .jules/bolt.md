## 2025-01-20 - Memoizing MessageRow
**Learning:** Found that `MessageRow` in `src/App.tsx` was not memoized, causing unnecessary re-renders of the entire message list whenever the chat state (like typing indicator, new stream chunks) updated. This is a common performance bottleneck in React chat applications.
**Action:** Applied `React.memo` (imported as `memo` from `react`) to `MessageRow` to prevent re-rendering of all historical messages when only the latest message or input state changes.
## 2025-05-17 - Memoizing Streaming Markdown Components
**Learning:** Found that multiple markdown parsing components (`TextBlock`, `ThoughtBlock`, `ToolBlock`, `MarkdownTable`, `MarkdownBody`) in `src/App.tsx` and `CodeBlock` in `src/components/CodeBlock.tsx` lacked `React.memo()`. Given that `MarkdownBody` renders token-by-token during streaming, non-memoized sub-components cause O(N^2) re-renders, recalculating static content like `split("\n")` or markdown parsing continuously. Additionally, `MarkdownTable` creates a new `lines` array prop every render and requires a custom memo comparison.
**Action:** Applied `React.memo()` to all static markdown components. Used a custom comparison function for `MarkdownTable` (`(prev, next) => prev.lines.join('\n') === next.lines.join('\n')`) to properly avoid re-renders. This ensures that only new components are rendered as the stream progresses, improving UI responsiveness.
