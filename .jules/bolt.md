## 2025-01-20 - Memoizing MessageRow
**Learning:** Found that `MessageRow` in `src/App.tsx` was not memoized, causing unnecessary re-renders of the entire message list whenever the chat state (like typing indicator, new stream chunks) updated. This is a common performance bottleneck in React chat applications.
**Action:** Applied `React.memo` (imported as `memo` from `react`) to `MessageRow` to prevent re-rendering of all historical messages when only the latest message or input state changes.

## 2025-02-12 - Inline functions break React.memo() during streaming
**Learning:** In chat applications, rendering a large list of `MessageRow` components that are wrapped in `React.memo()` can still suffer from O(N) re-renders if an inline function is passed as a prop (e.g. `onReact={(r) => reactToMessage(msg.id, r)}`). The parent component re-renders frequently (e.g. on every new token chunk during streaming), which creates a new reference for the inline function, thereby bypassing the memoization entirely. Furthermore, passing a global `streaming` boolean prop to all historical messages also breaks memoization of all messages when the streaming state toggles.
**Action:** Always ensure props passed to `React.memo` components are referentially stable. Use `useCallback` at the parent level, and delegate ID application to the child component (e.g. `onReact={reactToMessage}` and call `onReact(msg.id, reaction)` inside the child). Also, only pass volatile state like `streaming` to the specific component that needs it (e.g., `streaming={i === messages.length - 1 ? streaming : false}`).
